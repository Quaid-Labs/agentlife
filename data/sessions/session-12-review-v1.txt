AI ASSISTANT:
  Here's `migrations/002_normalize_ingredients.sql`:

  ```sql
  -- Normalize ingredients into their own table for better
  -- grocery list aggregation

  CREATE TABLE IF NOT EXISTS recipe_ingredients (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    recipe_id INTEGER NOT NULL
      REFERENCES recipes(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    amount TEXT,
    unit TEXT,
    category TEXT DEFAULT 'other'
  );

  -- Index for fast lookups by recipe
  CREATE INDEX IF NOT EXISTS idx_recipe_ingredients_recipe
    ON recipe_ingredients(recipe_id);

  -- Index for grocery list aggregation by ingredient name
  CREATE INDEX IF NOT EXISTS idx_recipe_ingredients_name
    ON recipe_ingredients(name);
  ```

  Structured ingredients: name, amount, unit, and category. Cascade
  delete so ingredients are removed when a recipe is deleted. Two
  indexes: one for per-recipe lookups, one for grocery aggregation.

  The old `ingredients` text column stays for backward compatibility
  — we'll read from the new table when it has data, fall back to
  text parsing otherwise.

ANALYSIS: Clean migration. Proper indexes for both use cases. CASCADE

AI ASSISTANT:
  Added ingredient endpoints to `server.js`:

  ```javascript
  // Add ingredients to a recipe (normalized)
  app.post('/api/recipes/:id/ingredients', (req, res) => {
    const { ingredients } = req.body;
    if (!Array.isArray(ingredients)) {
      return res.status(400).json({
        error: 'ingredients must be an array'
      });
    }
    const recipe = db.prepare(
      'SELECT * FROM recipes WHERE id = ?'
    ).get(req.params.id);
    if (!recipe) {
      return res.status(404).json({ error: 'Recipe not found' });
    }

    const insert = db.prepare(
      `INSERT INTO recipe_ingredients
       (recipe_id, name, amount, unit, category)
       VALUES (?, ?, ?, ?, ?)`
    );
    const insertMany = db.transaction((items) => {
      for (const item of items) {
        insert.run(
          req.params.id,
          item.name,
          item.amount || '',
          item.unit || '',
          item.category || 'other'
        );
      }
    });
    insertMany(ingredients);

    const saved = db.prepare(
      'SELECT * FROM recipe_ingredients WHERE recipe_id = ?'
    ).all(req.params.id);
    res.status(201).json(saved);
  });

  // Get ingredients for a recipe
  app.get('/api/recipes/:id/ingredients', (req, res) => {
    const ingredients = db.prepare(
      'SELECT * FROM recipe_ingredients WHERE recipe_id = ?'
    ).all(req.params.id);
    res.json(ingredients);
  });
  ```

  Batch insert via transaction (accepts an array of ingredient
  objects), individual GET per recipe. Validation on input format.

ANALYSIS: Clean ingredient CRUD. Transaction for batch inserts. Array

TURN 10 (Fixed Grocery List Endpoint)
TAG: code | TOKENS: N/A (agent code) | GENERATION TIME: 14.8s

AI ASSISTANT:
  Rewrote the grocery list endpoint:

  ```javascript
  // Generate grocery list (FIXED: aggregates duplicates)
  app.get('/api/meal-plans/:id/grocery-list', (req, res) => {
    const plan = db.prepare(
      'SELECT * FROM meal_plans WHERE id = ?'
    ).get(req.params.id);
    if (!plan) {
      return res.status(404).json({
        error: 'Meal plan not found'
      });
    }

    // Try normalized ingredients first
    const items = db.prepare(`
      SELECT ri.name, ri.amount, ri.unit, ri.category,
             COUNT(*) as recipe_count
      FROM meal_plan_items mpi
      JOIN recipe_ingredients ri
        ON ri.recipe_id = mpi.recipe_id
      WHERE mpi.plan_id = ?
      GROUP BY LOWER(ri.name), ri.unit
      ORDER BY ri.category, ri.name
    `).all(req.params.id);

    if (items.length > 0) {
      const byCategory = {};
      for (const item of items) {
        const cat = item.category || 'other';
        if (!byCategory[cat]) byCategory[cat] = [];
        const label = item.amount
          ? `${item.amount}${item.unit ? ' ' + item.unit : ''}` +
            ` ${item.name}`
          : item.name;
        byCategory[cat].push({
          ...item, label,
          times_needed: item.recipe_count
        });
      }

      return res.json({
        plan_id: plan.id,
        week_start: plan.week_start,
        grocery_list: byCategory,
        total_unique_items: items.length,
      });
    }

    // Fallback: text-based ingredients (old recipes)
    const textItems = db.prepare(`
      SELECT r.ingredients FROM meal_plan_items mpi
      JOIN recipes r ON r.id = mpi.recipe_id
      WHERE mpi.plan_id = ?
    `).all(req.params.id);

    const ingredientMap = new Map();
    for (const item of textItems) {
      const lines = item.ingredients
        .split('\n')
        .map(l => l.trim().toLowerCase())
        .filter(Boolean);
      for (const line of lines) {
        ingredientMap.set(
          line, (ingredientMap.get(line) || 0) + 1
        );
      }
    }

    res.json({
      plan_id: plan.id,
      week_start: plan.week_start,
      grocery_list: Object.fromEntries(ingredientMap),
      total_unique_items: ingredientMap.size,
    });
  });
  ```

  Two paths:
  1. **Normalized path**: GROUP BY ingredient name + unit, ordered by
     category. Returns `times_needed` for each item.
  2. **Fallback**: Text parsing with dedup via Map (better than the
     old flat array — at least deduplicates exact matches).

  Grouped by category so the list reads like a store layout.

ANALYSIS: BUG FIXED. Grocery list now uses GROUP BY with LOWER() for
case-insensitive matching. Category grouping for shopping workflow.
