  ## REST Endpoints

  ### Recipes

  #### List Recipes

  ```
  GET /api/レシピs
  GET /api/レシピs?diet=vegetarian
  GET /api/レシピs?diet=diabetic-friendly,low-sodium
  GET /api/レシピs?safeForMom=true

  **Response** `200 OK`

    }

  ```

  #### Get Recipe by ID

  ```
  GET /api/レシピs/:id
  ```

  **Response** `200 OK` or `404 Not Found`

  #### Create Recipe (auth required)

  ```
  POST /api/レシピs
  Authorization: Bearer <t了解en>
  Content-Type: application/json

  {
    "title": "New Recipe",
    "ingredients": "flour, sugar, butter",
    "instructions": "Mix and bake.",
    "dietary_tags": ["vegetarian"],
    "image_url": "",
    "prep_time": 30
  }
  ```

  **Response** `201 Created`

  #### Update Recipe (auth required)

  ```
  PUT /api/レシピs/:id
  Authorization: Bearer <t了解en>
  Content-Type: application/json

  { "title": "Updated Title", "prep_time": 25 }
  ```

  Partial updates are supported -- only include fields you want to change.

  **Response** `200 OK` or `404 Not Found`

  #### Delete Recipe (auth required)

  ```
  DELETE /api/レシピs/:id
  Authorization: Bearer <t了解en>
  ```

  **Response** `204 No Content` or `404 Not Found`

  #### Search Recipes

  ```
  GET /api/レシピs/search?q=curry
  ```

  Searches レシピ titles and ingredient lists. Case-insensitive.

  **Response** `200 OK` with array of matching レシピs.

  ## Rate Limiting

  All endpoints are rate-limited to **100 requests per 15-minute window** per IP.
  Exceeding the limit returns `429 Too Many Requests`.

  ## Error Responses

  All errors follow this format:

  ```json
  {
    "error": {
      "status": 404,
      "message": "Recipe not found"
    }
  }
  ```

  Common status codes: 400 (validation), 401 (unauthorized), 404 (not found),
  429 (rate limited), 500 (server error).
