TURN 10 (SQL Injection Discovery)
TAG: normal | TOKENS: 87/100 | GENERATION TIME: 8.5s

AIアシスタント:
  ねえ、再設計のために検索エンドポイントを更新している途中で、
  ひとつ注意したいことを見つけた。

  検索関数にSQLインジェクションの脆弱性がある。`server.js` の
  この行を見て:

  ```javascript
  // BEFORE (vulnerable):
  const recipes = db.prepare(
    "SELECT * FROM recipes WHERE name LIKE '%" + q + "%' OR ingredients LIKE '%" + q + "%'"
  ).all();
  ```

  この `q` の値が、サニタイズなしでSQL文字列にそのまま入っている。
  仕組みのある検索文字列を送れば、データベースをおかしくできる可能性がある。
  他のエンドポイントはどれもパラメータ化クエリを正しく使っているのに、
  ここだけ抜けていたんだ。

  修正はこう:

  ```javascript
  // AFTER (safe):
  app.get('/api/search', (req, res) => {
    const { q } = req.query;
    if (!q) return res.json([]);
    const recipes = db.prepare(
      "SELECT * FROM recipes WHERE name LIKE ? OR ingredients LIKE ?"
    ).all(`%${q}%`, `%${q}%`);
    res.json(recipes);
  });
  ```

  これで検索語はプリペアドステートメントのパラメータを通る。ほかの
  エンドポイントと同じやり方だ。

マヤ:
  え、ちょっと待って。ずっと壊れたままだったの？

  つまり、誰かが私のデータベースをいじれたってこと？
